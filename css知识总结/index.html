浏览器的渲染原理

渲染流程:

  • 根据 html 生成 DOM 节点树
  • 根据 CSS 生成 CSSOM
  • 合并 html 和 CSS，生成待渲染的树
  • layout 确定布局，计算大小和位置
  • paint 绘制上色、阴影等
  • composite 合并图层，根据层叠关系展示画面

画面更新流程：

1、layout => paint => composite

2、paint => composite

3、composite

三个流程，不同属性、不同浏览器有不同，流程越少性能越好，具体查csstriggers.com

CSS 动画的两种做法

transition

作用是两个关键帧之间添加过渡效果，限制是只能两个关键帧，也可以通过监听中间节点来达到多次变化，但是麻烦。

animation

通过@keyframes可以设置多个关键帧，还可以定义循环次数，方向，暂停等等，功能强大。

  • 学完这课就是把基本的常用动画过了一遍，详细的还要看 MDN 文档，还有就是要多写，记住一些常用的例子以及一些坑。
